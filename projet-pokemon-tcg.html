Jeu Pokémon TCG en Java

Projet académique de 1ère année de BUT Informatique, centré sur la programmation orientée objet et la modélisation UML.

Détails du projet

Contexte :Projet pédagogique BUT1
Durée :4 à 5 semaines
Format :Travail en équipe (2–3 personnes)

Contexte du projet

L’objectif était de développer un jeu de cartes à collectionner inspiré de Pokémon TCG en Java, en appliquant les principes de la POO (encapsulation, héritage, polymorphisme) et en partant d’une modélisation UML.

  • Commande académique dans le cadre d’une SAE de programmation orientée objet.
  • Production attendue : application jouable en console avec gestion de decks et de combats.
  • Contraintes : structure orientée objet claire, classes cohérentes, code documenté.

Méthodes de travail et résultats

Organisation et approche :

  • Réalisation d’un diagramme de classes UML avant toute implémentation.
  • Utilisation de Git pour le suivi des versions et la répartition des tâches.
  • Découpage en modules : gestion des cartes, gestion des decks, moteur de combat.
Fonctionnalités principales :
  • Classes de>Carte, de>CartePokemon, de>Deck, de>Joueur.
  • Système de points de vie, attaques et calcul de dégâts.
  • Gestion d’un deck de cartes, tirage, main du joueur.
  • Interface en ligne de commande avec menus de choix.

Le livrable final est un jeu jouable en console, accompagné d’une Javadoc et de quelques tests unitaires JUnit.

Compétences travaillées

Techniques :

Java – POO UML (diagramme de classes) Gestion de collections Tests unitaires (JUnit) Git / GitLab

Humaines :

Travail en équipe Organisation du travail Communication technique
